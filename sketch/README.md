# スケッチ

このディレクトリは甲州記法の使用例を束ねたものです。
スケッチのように詳しい説明をつけずに、コードの実行例だけを保存しています。
使用例やテストケースの作成を簡素化し、
時間をかけずに多くの例をつくることを目的としています。
スケッチの一覧は [入出力リストの要約] をみてください。

新しいスケッチを作成するときは、つぎのようにします。
この説明のなかの `foo` は適当な名前に読み替えてください。

 1. 甲州記法の使用例をファイル [`foo.k`](foo/foo.k)
    に保存し、`koshu foo.k` で実行します。

 2. それをスケッチとして保存したいときは、
    ディレクトリ [`foo`](foo) をつくり、そのなかに `foo.k` を保存します。

 3. 実行例を入出力リストとして文書化するため、ディレクトリ `foo`
    のなかで `koshu-inout-init.sh -r` を実行し、
    [`INOUT.sh`](foo/INOUT.sh) を生成します。
    `-r` オプションは、入出力リストの保存先を `README.md` に変更します。

 4. その `./INOUT.sh` を実行し、入出力リスト
    [`README.md`](foo/README.md) を生成します。

 5. `sketch` ディレクトリにもどって `koshu-inout-summary.sh` を実行し、
    追加されたスケッチを含めた全体の退行テストをします。

 6. その結果に問題がなければ、`koshu-inout-summary.sh -ls`
    を実行して、[入出力リストの要約] を更新します。
    `-ls` オプションでリンクつきの要約を保存するように指示します。

 7. 修正の必要がないか各ファイルを見直し、コミットします。

[入出力リストの要約]: INOUT-SUMMARY.md
